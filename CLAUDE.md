# BMS ELSA

## プロジェクト概要・構造
- `README.md`参照

## Gitブランチ運用ルール
- 設計ドキュメント（`*-design.md`）を`docs/`配下に作成する場合は、先にフィーチャーブランチを作成・切り替えてからコミットする
- 以降の実装作業もそのブランチ上で継続する（mainに直接コミットしない）

## ビルド
- コンパイル確認には `go build ./...` を使う（`go build .` はルートにバイナリを出力してしまうため不可）

## フロントエンド
- UI実装時は `docs/style-guide.md` のデザイン規約に従うこと
