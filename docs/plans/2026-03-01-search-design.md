# 各画面への検索機能追加 設計ドキュメント

## 概要

3つのタブ（songs/charts/difficulty）それぞれに検索バーを追加。
各テーブルコンポーネント内のヘッダー部にテキストフィールドを1つ配置し、タイトル・アーティスト等を横断検索する。

## 画面別の検索方式

| 画面 | 検索方式 | 理由 |
|------|---------|------|
| SongTable | バックエンド（既存`search`パラメータ活用） + debounce 300ms | ページネーション制御のため全件がメモリにない |
| ChartListView | フロントエンド（Tanstack Table `getFilteredRowModel()`） | 全件メモリ上。即座にフィルタ可能 |
| DifficultyTableView | フロントエンド（配列`.filter()`） | 選択テーブル内の全エントリがメモリ上 |

## UI配置

各コンポーネントの既存ヘッダー部（件数表示の行）に検索フィールドを追加:

```
┌──────────────────────────────────────┐
│ [タブバー: 楽曲一覧 / 譜面一覧 / 難易度表]  │
├──────────────────────────────────────┤
│ 1234 楽曲  [🔍 検索..._______________] │  ← ヘッダー行に検索追加
├──────────────────────────────────────┤
│ Title ▲ | Artist | Genre | BPM ...   │  ← テーブルヘッダー
│ ─────────────────────────────────── │
│ 曲A      | ...                       │
│ 曲B      | ...                       │
├──────────────────────────────────────┤
│ [詳細ビュー]                          │
└──────────────────────────────────────┘
```

## イベント管理（deselect問題の解決）

検索バーの行全体に`on:click|stopPropagation`を付与。

- 検索フィールドのクリック/フォーカス → 伝播停止 → 詳細ビューは閉じない
- テーブル行クリック → 既存通り`stopPropagation` + `select`イベント
- テーブル空白領域クリック → 既存通り`deselect`イベント発火

## 検索対象フィールド

| 画面 | 検索対象 |
|------|---------|
| SongTable | title, artist, genre（既存バックエンド実装通り） |
| ChartListView | title, subtitle, artist, subArtist, genre |
| DifficultyTableView | title, artist |

## 検索時の挙動

- 検索文字列変更時、選択状態（詳細ビュー）はクリアしない
- テーブル切り替え（DifficultyTableView）時は検索文字列をクリア
- タブ切り替え時、各タブの検索状態は独立で保持
