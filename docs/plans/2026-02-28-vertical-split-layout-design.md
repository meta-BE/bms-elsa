# 上下分割レイアウト

## 目的

詳細パネルを右側から画面下部に移動し、テーブルが全幅を使えるようにする。
副次効果として、TITLE列の左端が切れる問題が解消される。

## 設計

### レイアウト

```
┌─────────────────────────────┐
│  navbar (BMS ELSA)          │
├─────────────────────────────┤
│                             │
│  SongTable (上部)            │
│                             │
├═══ ドラッグハンドル ═════════┤
│  SongDetail (下部)           │
│                             │
└─────────────────────────────┘
```

- 詳細パネル未選択時はテーブルが全高を使用
- デフォルト比率: 上60% / 下40%

### ドラッグリサイズ

- 境界線上に4pxのドラッグ可能エリア
- カーソル: `cursor-row-resize`
- リサイズ位置の永続化はしない

### パネルの開閉

- **行クリック** → その行の詳細ビューを表示（別の行ならビュー切替）
- **カラムヘッダークリック** → ソート（現状通り）
- **テーブル空き部分クリック** → 詳細パネルを閉じる（行・ヘッダー以外の領域）
- **閉じるボタン** → 詳細パネル右上の×ボタン

テーブルのスクロールコンテナに click ハンドラーを設定し、行・ヘッダーからのクリックは stopPropagation() で伝播を止める。

### 変更ファイル

- `App.svelte` — flex-row → flex-col、ドラッグリサイズロジック追加
- `SongTable.svelte` — 空き部分クリックで閉じるイベント追加
- `SongDetail.svelte` — 閉じるボタン追加

### やらないこと

- リサイズ位置の永続化（localStorage等）
- 開閉アニメーション
- 最小/最大サイズ制約
