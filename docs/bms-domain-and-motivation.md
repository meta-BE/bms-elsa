# BMSドメイン知識とソフトウェアのモチベーション

## BMSフォルダ構成

1フォルダが1曲を表す。

```
<曲フォルダ>/
├── *.wav, *.ogg       # こまぎれの音声群（「音」）
├── *.bms, *.bme, *.bml # 譜面ファイル（「音」の配置を定義し、映像と合わせて再生する）
└── *.mp4, *.wmv, *.bmp # 映像・画像（連続表示で擬似映像になることもある）
```

## BMSファイルの構造

### ヘッダ部（メタ情報）

| フィールド | 説明 | 例 |
|---|---|---|
| `#TITLE` | 曲名。暗黙的にサブタイトルを含む場合がある | `Act #000000 (short)` |
| `#SUBTITLE` | 明示的なサブタイトル定義 | `[Extra]` |
| `#ARTIST` | アーティスト名 | `Saz` |
| `#SUBARTIST` | 補助クレジット（差分作者、ボーカル、映像作者など） | `obj: qtp13` |
| `#GENRE` | ジャンル | `trance` |
| `#BPM` | テンポ（小数対応） | `150` |
| `#PLAYLEVEL` | 数値的な難易度 | `2`, `24` |
| `#DIFFICULTY` | 難易度カテゴリ（1=EASY, 2=NORMAL, 3=HARD, 4=EXTRA, 5=INSANE） | `5` |
| `#RANK` | 判定窓の厳しさ（0=VERY HARD〜3=EASY） | `2` |
| `#TOTAL` | ゲージ増加量。デフォルト値は実装依存（LR2=160） | `353` |
| `#STAGEFILE` | ロード画面の画像（640x480） | `trance.bmp` |
| `#BANNER` | 選曲画面用バナー画像（300x80） | `banner.bmp` |

- `#PLAYER` はLR2/beatorajaでは無視され、使用チャンネルからSingle/Doubleが推定される
- `#SUBARTIST` は差分作者だけでなく、ボーカル・映像作者などのクレジットとして本家譜面にも付与される
- 仕様リファレンス: https://hitkey.nekokan.dyndns.info/cmdsJP.htm

### 定義部

- `#WAVxx <ファイル名>` — 音声ファイルを16進番号に割り当て
- `#BMPxx <ファイル名>` — BGA用画像/映像ファイルを16進番号に割り当て

1譜面あたり数十〜数百の定義を持つ。

### データ部

`#xxxyy:data` 形式で譜面データを記述。

- `xxx` = 小節番号
- `yy` = チャンネル番号（`01`=BGM, `11`〜`18`=鍵盤, `04`/`07`=BGA等）
- `data` = 16進数2桁ペアでオブジェクト配置

## 外部メタデータソース: LR2IR

LR2IR（http://www.dream-pro.info/~lavalse/LR2IR/）にBMS譜面のメタデータが集約されている。

### アクセス方法

- bmsid指定: `search.cgi?mode=ranking&bmsid=<ID>`
- **MD5指定**: `search.cgi?mode=ranking&bmsmd5=<BMSファイルのMD5>`

BMSファイルのMD5ハッシュでアクセス可能なため、ローカルのBMSファイルから直接メタデータを引ける。

### LR2IRの特性

- **譜面単位**（.bmsファイル単位）でエントリが分かれる。同じ曲フォルダ内の譜面でも別エントリになる
- ARTIST表示は `#ARTIST` + `#SUBARTIST` の合成値（例: `Saz obj: qtp13`）
- 取得可能な情報: タイトル、アーティスト、ジャンル、BPM、難易度、タグ、本体URL、プレイ統計など

## 差分の正当性検証

差分ファイル（.bms/.bme/.bml）が正しい曲フォルダに配置されているかの検証には、以下の手段がある。

1. **メタデータの一致確認** — `#TITLE`, `#ARTIST` のベース部分が既存譜面と一致するか
2. **参照ファイルの存在確認** — `#WAVxx`, `#BMPxx` で定義されたファイルがフォルダ内に存在するか
3. **LR2IRでの照合** — MD5ハッシュを使ってLR2IR上のメタデータを取得し、同一曲であることを確認

特に参照ファイルの存在確認は、メタデータが不十分な場合でも有効な検証手段となる。差分譜面は同じ曲の音声・映像素材を再利用するため、正しいフォルダに配置されていれば大半のファイルが存在するはず。

## 主要な操作

### 楽曲の導入

曲フォルダを適切なディレクトリに配置する作業。

### 差分の導入（譜面の導入）

.bms/.bme/.bmlファイルを、対応する曲フォルダに移動する作業。
誤った曲フォルダに配置すると正しく再生できなくなるため、正確なマッチングが必要。

## 現状の課題

1. **フォルダ名による曲の識別が困難** — 大量のフォルダがあり、フォルダ名だけでは曲を特定しづらい
2. **差分の誤配置リスク** — 間違えると再生不能になる
3. **ディレクトリの肥大化** — 1ディレクトリに大量の曲フォルダがあるとWindowsエクスプローラーが重くなる
4. **分類の手間** — 適切に分類するにはBMSファイル内のメタ情報の解析やWebサイトでの検索が必要

## ソフトウェアのモチベーション

楽曲導入・差分導入を、BMSファイルのメタ情報をベースになるべく自動で行えるようにする。
