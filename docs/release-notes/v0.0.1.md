## 新機能
- BMS 楽曲・譜面メタデータ管理アプリの初回リリース。Wails v2 + Svelte 4 + Go によるデスクトップアプリの基盤を構築。
- beatoraja の `songdata.db` を ATTACH して楽曲一覧・譜面一覧・楽曲詳細を表示する機能を追加。フォルダ単位での楽曲グルーピングと、譜面ごとの subtitle/subartist 表示にも対応。
- LR2IR からのメタデータ取得機能を追加。HTTP クライアント・HTML パーサー・レートリミットを備え、`elsa.db` の `song_meta` / `chart_meta` テーブルにキャッシュ保存する。
- 楽曲一覧タブ・譜面一覧タブ・難易度表タブを備えたメイン UI を実装。タブ切り替え時の状態保持、上下分割レイアウト（ドラッグリサイズ）、楽曲詳細・譜面詳細・難易度表エントリ詳細パネルを提供。
- BMS 難易度表の取り込み機能を追加。HTML/JSON フェッチャー、難易度表テーブル管理 UI（追加・削除・更新）、難易度バッジ表示、難易度表エントリ一覧ビュー（EntryDetail）に対応。
- 各画面（楽曲一覧・譜面一覧・難易度表）に検索フィールドを追加。楽曲一覧は全件取得＋フロントエンドフィルタに統一し、TanStack Table の globalFilter で検索する。
- 設定画面モーダルを追加。`songdata.db` のパス指定、難易度表管理、ファイル選択ダイアログに対応。`config.json` / `elsa.db` は実行ファイル隣接配置に統一。
- タグ push 時に Windows 向けバイナリを自動ビルドし GitHub Release を作成する CI を追加。
- 負荷試験用に 10,000 曲規模の `songdata_10k.db` を生成するスクリプト (`cmd/gen-testdata`) を追加。

## バグ修正
- 譜面ごとの BPM 演出（ギミック）により全譜面の MIN/MAX で不正確な値になる問題を、(minbpm, maxbpm) ペアの最頻値で代表 BPM 範囲を表示するよう修正。
- `songdata.song.folder` カラムにインデックスを追加し、`ListSongs` の初回表示を 2.4 秒から 0.02 秒に高速化（2,666 フォルダ・15,437 譜面環境）。
- ドラッグリサイズ終了時のクリックイベントで詳細パネルが閉じてしまう問題を修正。
- テーブル外の余白クリックでも詳細パネルが閉じるよう修正。同じ行の再クリックで詳細パネルを閉じるトグル動作も追加。
- ChartListView のヘッダー幅ずれと行クリック不動作を修正。テーブルヘッダーとボディのカラム幅を揃えるため `<table>` から `<div class="flex">` ベースに変更。
- 検索クリアボタンのレイアウトずれを修正し、全テーブルの選択状態を props 駆動に統一して deselect 時のハイライト残りを解消。
- 難易度表 level カラムを数値順ソートに修正。
- `UpsertChartMeta` の ON CONFLICT で working URL が上書きされる問題を修正し、`UpdateWorkingURLs` を分離して LR2IR データを保持するよう設計変更。
- A11y 警告（スクロール領域の role/keydown、ドラッグハンドル）を修正。
