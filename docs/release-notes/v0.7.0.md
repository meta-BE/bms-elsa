## 新機能
- BMS Search連携によるイベント情報管理を導入。従来のURLパターンマッチング（event_mapping）方式を廃止し、BMS Search APIから楽曲とイベントの紐付けを自動取得する方式に刷新した。LR2IRのURL未取得の楽曲にもイベント情報を設定できるようになった。
- イベントマスター管理画面（EventManager）を新設。BMS Searchからのイベント取得、短縮名のインライン編集、年・正式名称・短縮名のカラム表示に対応。
- 楽曲一覧にBMS Search同期ボタンを追加。フォルダ単位で進捗表示・停止操作・並列同期（3並列）に対応し、同期済みフォルダは再同期対象から除外する。
- 楽曲詳細にイベントのオートコンプリートセレクターを追加。イベント変更時に楽曲一覧の該当行がリアルタイムで更新される。
- 楽曲詳細にBMS Searchページ・イベントページ（exhibitions/{event_id}）への外部リンクを追加。
- 難易度表の一括更新を非同期化し、最大5テーブル並列で取得するようにした。設定ダイアログに進捗表示・停止ボタンを追加。
- 難易度表メインUIに個別更新ボタン（arrowPathアイコン）を追加。

## バグ修正
- 個別更新の完了メッセージで成功・失敗に応じて表示色を使い分けるよう修正。
- 設定ダイアログ再オープン時に進捗表示が復元されない問題を修正（dt:refresh-doneガード追加とopen時の状態復元）。
- 一括更新完了時にcontextリソースが解放されない問題を修正。
