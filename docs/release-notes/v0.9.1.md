## 新機能
- 差分導入画面のテーブルを TanStack Table + 仮想スクロールに移行し、ヘッダードラッグによるカラム幅リサイズに対応。リサイズはハンドル境界の左右カラムを同時に調整する方式とし、合計幅が常に保たれてビューポートからはみ出さないようにした。
- ソート可能ヘッダー（SortableHeader）共通でリサイズハンドルを表示できるようにし、他のテーブルでも今後利用できる基盤を整備した。

## バグ修正
- ヘッダーの `min-width: auto` によりカラムがリサイズで最小幅以下に縮まない問題を修正。
- リサイズ操作時に隣接以外のカラムまで連動して動いてしまう問題を修正（コンテナ幅を測定して flex カラムの実ピクセル幅をロックすることで対処）。
- 差分導入画面で同一ファイルパスのドロップによる重複追加を防止し、クリア時に全件削除されてしまう問題を回避。
