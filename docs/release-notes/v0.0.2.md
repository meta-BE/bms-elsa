## 新機能
- 楽曲メタデータ推測機能を追加。LR2IR 本体 URL のパターンマッチで楽曲の ReleaseYear / EventName を半自動設定する。`event_mapping` テーブルでパターン→イベント名・年を管理し、自動マッチ→未マッチ曲の手動確認の 2 フェーズ UI を提供。BOF 系・manbow・PABAT・venue・ゆるゆるなど 214 件のマッピングを CSV で初期投入。
- IR 情報の一括取得機能を追加。譜面一覧および難易度表ビューから未取得譜面の IR 情報をまとめてフェッチでき、開始・停止が可能。未導入譜面でも `chart_meta` から IR 情報を表示できるよう EntryDetail を改善。
- URL 書き換えルール機能を追加。LR2IR の本体 URL に対するルール管理モーダル、SongTable / ChartListView での動作 URL 推定ボタン、SongDetail / ChartDetail / EntryDetail での動作 URL リンク表示と編集ボタンを提供。
- 重複検知機能を追加。タイトル・アーティスト・ジャンルのファジー類似度と BPM 範囲重複を用いて重複楽曲を検出する専用タブ（DuplicateView / DuplicateDetail）を実装し、グループ選択時に各メンバーの譜面（subtitle + ファイル名）を表示する。
- カラムフィルタ機能を追加。SongTable / ChartListView の EVENT・YEAR、DifficultyTableView の STATUS をフィルタドロップダウンメニューに変更。`meta.filterSort` で選択肢のソート順を指定でき、YEAR は降順表示。
- テーブル上下キーナビゲーションを追加。3 つのテーブル全てで矢印キー操作が可能になり、EntryDetail を md5 + tableID で自己取得するパターンに統一。
- 外部リンクをシステムブラウザで開く機能を追加し、Wails の制限を回避。クロスプラットフォーム（macOS / Windows / Linux）対応。
- リリース用 Makefile を追加。`make release-patch/minor/major` でセマンティックバージョンタグを自動インクリメント・push し、CI ビルドをトリガーする。

## バグ修正
- メタ推測の `ListUnsetSongsWithIRURLs` が常に 0 件を返すバグを修正。`songdata.folder` テーブルとの JOIN 条件が誤っていたため、他クエリ同様に `song.folder` カラムを直接使うよう変更。
- メタ推測モーダルを閉じた後に SongTable を再読み込みするよう修正。LR2IR リクエスト間隔を 1 秒から 500ms に短縮。
- ATTACH スキーマ名が `sd` のままの箇所を `songdata` に修正。対象 0 件時の表示も改善。
- IR 一括取得 UI を SongTable から ChartListView に移動。
- `chart_meta` の PK を md5 のみに変更し、`ChartKey` を削除。`UpdateChartMeta` から sha256 パラメータを削除。
- LR2IR の本体 URL（`~lavalse` 等のチルダ含む）がシェルメタ文字検証で開けない問題を、Go 側の `os/exec.Command` で `open` コマンドを直接呼ぶことで回避。
- `SortableHeader` のテンプレート内キャストをヘルパー関数に修正。
